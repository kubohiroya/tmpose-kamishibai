TMPose紙芝居(かみしばい) 利用(りよう)・台本作成(さくせい)ガイド

このHTMLとPDFは、小学3年生までに学ぶ漢字を既習(きしゅう)として、以後(いご)に学ぶ漢字へふりがなを付(つ)けています。

TMPose紙芝居(かみしばい)は、物語の登場人物と同じポーズを取ることで次の場面へ進む、参加(さんか)型(がた)のデジタル紙芝居(かみしばい)です。TurboWarpで作られた紙芝居(かみしばい)プレイヤー、TMPoseによるカメラ映像(えいぞう)のポーズ認識(にんしき)、場面を記述(きじゅつ)するテキスト形式の台本を組み合わせて使います。

このガイドは、はじめて作品を上演する人と、新しい台本を作る人のための説明書です。公開中のWeb版を開きながら読むと、操作を確かめやすくなります。PDF版をダウンロードして印刷することもできます。

1. できること

  • 背景(はいけい)、登場人物、せりふ、音楽、効果(こうか)音を場面ごとに再生(さいせい)できます。
  • カメラの前で指定されたポーズを取ると、物語が進みます。
  • 紙芝居(かみしばい)の内容(ないよう)はテキスト台本で管理(かんり)するため、プログラム本体を変更(へんこう)せずに作品を作れます。
  • TurboWarpでSB3ファイルを開き、画像(がぞう)、音、ポーズ認識(にんしき)先などを編集(へんしゅう)できます。

必要(ひつよう)なもの

用意するもの 説明(せつめい)
端末(たんまつ) カメラを使えるパソコン、タブレット、またはスマートフォン
ブラウザ 新しいバージョンのChromeなど、カメラ利用(りよう)を許可(きょか)できるブラウザ
紙芝居(かみしばい)プレイヤー 公開中のWeb版(ばん)、またはTurboWarpで開いたSB3ファイル
台本 UTF-8で保存(ほぞん)したテキストファイル
TMPoseモデル 台本のTMPoseURLから読(よ)み込(こ)む、ポーズ学習済(ず)みページ

2. 上演(じょうえん)する

  1. Web版(ばん)を開きます。
  2. ブラウザから確認(かくにん)されたら、カメラの利用(りよう)を許可(きょか)します。
  3. 開くボタンを押(お)し、紙芝居(かみしばい)の台本ファイルを選(えら)びます。
  4. 表紙と題名を確認(かくにん)し、開始操作(そうさ)をします。
  5. 「ポーズを取ろう」という案内(あんない)が出たら、カメラに全身が映(うつ)る位置(いち)へ移動(いどう)します。
  6. 画面の見本と同じポーズを取り、認識(にんしき)が完了(かんりょう)するまで姿勢(しせい)を保(たも)ちます。
  7. 最後(さいご)の場面まで進んだら、再(さい)読(よ)み込(こ)みボタンで同じ作品をもう一度始められます。

カメラを使うときの注意

  • 周(まわ)りの人や物にぶつからないよう、両手を広げられる場所を確保(かくほ)してください。
  • 端末(たんまつ)は倒(たお)れない場所へ置(お)き、カメラに頭から足まで映(うつ)る距離(きょり)を取ってください。
  • 明るい場所で、背景(はいけい)と服の色が大きく異(こと)なると認識(にんしき)しやすくなります。
  • カメラ映像(えいぞう)はポーズ判定(はんてい)に使います。利用(りよう)するWeb版(ばん)とTMPoseモデルの管理(かんり)者、通信(つうしん)先、保存(ほぞん)方針(ほうしん)を上演(じょうえん)前に確認(かくにん)してください。

うまく認識(にんしき)されないとき

  1. ブラウザのアドレス欄(らん)にあるカメラ表示(ひょうじ)を確認(かくにん)し、利用(りよう)を許可(きょか)します。
  2. 体が画面から切れていないか確認(かくにん)します。
  3. 見本と左右が反対に見える場合は、画面を鏡(かがみ)として同じ形になるよう試(ため)します。
  4. 別(べつ)の人や家具が体と重ならないようにします。
  5. 改善(かいぜん)しない場合は、ページを再(さい)読(よ)み込(こ)みして台本を開き直します。

3. 台本の全体像(ぞう)

台本は、作品全体の設定(せってい)と複数(ふくすう)の場面から構成(こうせい)されます。行頭が#の行はコメントです。場面と場面の間は---だけの行で区切ります。

kamishibai=3.1
# title: 浦島太郎
# author: Hiroya Kubo
setRuntimeVariable=startSceneIndex:1

asset=Beach1,backdrop
asset=Ocean Wave,sound
asset=Urashima-walk-1,costume:Urashima
actor=Urashima,Urashima-walk-1
cover=Beach1,Ocean Wave
---
# scene 1
sceneLabel=beach
TMPoseURL=https://example.org/tmpose/model/
action=stage:Beach1
action=Urashima:show:Urashima-walk-1:0,-60,45
action=Urashima:say:今日は浜辺を散歩しよう。:2
action=wait:1

書き方の基本(きほん)

  • 文字コードはUTF-8にします。
  • 基本形(きほんけい)はキー=値です。=の左右には余分(よぶん)な空白を入れません。
  • actionの値(ね)は:で区切ります。引数の個数(こすう)と順序(じゅんじょ)はコマンドごとに異(こと)なります。
  • 画像(がぞう)名、音名、登場人物名は大文字と小文字、空白を含(ふく)めて素材(そざい)側(がわ)の名前と合わせます。
  • 時間は秒、座標(ざひょう)はScratchのステージ座標(ざひょう)、サイズは百分率(ひゃくぶんりつ)で指定します。
  • 空のsayやthinkを実行すると、表示(ひょうじ)中の吹(ふ)き出(だ)しを消せます。

4. 作品全体の設定(せってい)

設定(せってい) 形式 役割(やくわり)
バージョン kamishibai=3.1 台本形式のバージョンを示(しめ)します。最初(さいしょ)の行に置(お)きます。
実行時変数(へんすう) setRuntimeVariable=名前:値 開始場面など、実行時の変数(へんすう)を設定(せってい)します。
素材(そざい) asset=素材名,種類[:対象] プレイヤーから利用(りよう)する画像(がぞう)や音を登録(とうろく)します。
登場人物 actor=人物名,初期コスチューム 登場人物と最初(さいしょ)の見た目を登録(とうろく)します。
表紙 cover=背景名,音名 表紙に使う背景(はいけい)と音を指定します。
分岐(ぶんき) registerBranch=分岐名:条件一覧:場面ラベル一覧 条件(じょうけん)に応(おう)じた移動(いどう)先を登録(とうろく)します。

素材(そざい)の種類(しゅるい)

asset=Beach1,backdrop
asset=Ocean Wave,sound
asset=Turtle,costume
asset=Urashima-help-1,costume:Urashima

backdropは背景(はいけい)、soundは音、costumeは登場人物の見た目です。costume:Urashimaのように対象(たいしょう)を付(つ)けると、その登場人物の素材(そざい)として登録(とうろく)します。

5. 場面を作る

各(かく)場面には、移動(いどう)先として使うsceneLabel、必要(ひつよう)に応(おう)じたTMPoseURL、上から順(じゅん)に実行するactionを記述(きじゅつ)します。

---
# scene 2
sceneLabel=ocean
TMPoseURL=https://example.org/tmpose/ocean/
action=stage:Ocean
action=wait:1.5
action=Urashima:show:Urashima-ride-1:170,5,40
action=Urashima:moveTo:-100,-5,2

同じTMPoseモデルを使う場面でも、必要(ひつよう)な場面ごとにTMPoseURLを書くと台本を読み返しやすくなります。URLを切(き)り替(か)える場合は、モデルの読(よ)み込(こ)み時間を上演(じょうえん)計画に含(ふく)めてください。

6. ステージ用アクション

コマンド 例(れい) 動作
背景(はいけい) action=stage:Beach1 指定した背景(はいけい)へ切(き)り替(か)えます。
待機(たいき) action=wait:1.5 指定した秒数だけ待ちます。
BGM action=bgm:Guitar Chords2 音をBGMとして再生(さいせい)します。次のBGMで切(き)り替(か)わります。
効果(こうか)音 action=sound:Gong 指定した音を最後(さいご)まで再生(さいせい)します。
転換(てんかん) action=transition:fadeOut 画面を暗くします。fadeUpで明るくし、resetで効果(こうか)を戻(もど)します。
分岐(ぶんき) action=branch:ending 登録(とうろく)済(ず)みの条件(じょうけん)を評価(ひょうか)し、次の場面ラベルを決めます。
キー選択(せんたく) action=keyInputToChangeScene:KeyA,KeyB:route-a,route-b キー入力に応(おう)じて次の場面を選(えら)びます。
タッチ選択(せんたく) action=touchInputToChangeScene:ActorA,ActorB:route-a,route-b 触(ふ)れた登場人物に応(おう)じて次の場面を選(えら)びます。

7. 登場人物用アクション

登場人物用アクションはaction=人物名:命令:引数の形です。

命令(めいれい) 例(れい) 動作
表示(ひょうじ) action=Urashima:show:Urashima-walk-1:0,-60,45 見た目、座標(ざひょう)、サイズを指定して表示(ひょうじ)します。初期(しょき)の見た目を使う場合は見た目を省略(しょうりゃく)できます。
非(ひ)表示(ひょうじ) action=Urashima:hide 登場人物を隠(かく)します。
せりふ action=Urashima:say:こんにちは。:2 せりふを表示(ひょうじ)します。最後(さいご)の値(ね)は表示(ひょうじ)秒数です。
考えごと action=Urashima:think:どうしよう。:2 考えごとの吹(ふ)き出(だ)しを表示(ひょうじ)します。
見た目 action=Urashima:setSkin:Urashima-dance-1:45 コスチュームを変更(へんこう)します。サイズは省略(しょうりゃく)できます。
移動(いどう) action=Urashima:moveTo:120,-77,1.5 座標(ざひょう)へ移動(いどう)します。最後(さいご)の値(ね)は移動(いどう)秒数です。
ポーズ action=Urashima:pose:見た目一覧:ラベル一覧:音一覧 見本を順(じゅん)に示(しめ)し、各(かく)ポーズが認識(にんしき)されるまで待ちます。
反復(はんぷく) action=Fish:loop:Fish1,Fish2:1,1 複数(ふくすう)の見た目を、指定した秒数で繰(く)り返(かえ)します。
一回再生(さいせい) action=Fish:sequence:Fish1,Fish2:0.5,0.5 複数(ふくすう)の見た目を、指定した秒数で一回ずつ再生(さいせい)します。

show、say、think、setSkin、moveToの細かな省略(しょうりゃく)形は、使うSB3ファイルのバージョンによって異(こと)なる場合があります。新しい作品では上の例(れい)のように必要(ひつよう)な値(ね)を明示(めいじ)し、短い場面で動作確認(かくにん)してください。

8. ポーズ認識(にんしき)を組(く)み込(こ)む

poseでは、コスチューム名、TMPoseのポーズラベル、成功(せいこう)時の音をそれぞれカンマ区切りで並(なら)べます。三つの一覧(いちらん)は同じ個数(こすう)にします。

action=Urashima:pose:Urashima-dance-1,Urashima-dance-2:dance1,dance2:Drum Funky,Drum Funky

この例(れい)は次の順(じゅん)に進みます。

  1. Urashima-dance-1を見本として表示(ひょうじ)し、TMPoseのdance1を待ちます。
  2. 認識(にんしき)できたらDrum Funkyを鳴らします。
  3. Urashima-dance-2へ切(き)り替(か)え、dance2を待ちます。
  4. 二つ目も認識(にんしき)できたら次のactionへ進みます。

TMPoseモデルを準備(じゅんび)する目安

  • 各(かく)ラベルには、立つ位置(いち)、体の向き、腕(うで)や足の形が少し異(こと)なる複数(ふくすう)の例(れい)を学習させます。
  • 通常(つうじょう)姿勢(しせい)や関係(かんけい)のない姿勢(しせい)を判別(はんべつ)するためのラベルも用意します。
  • 上演(じょうえん)する部屋、端末(たんまつ)、参加(さんか)者で事前に試(ため)し、誤(あやま)認識(にんしき)しやすいポーズを調整します。
  • 台本のラベルとTMPose側(がわ)のラベルは、英字(えいじ)の大文字と小文字まで一致(いっち)させます。

9. 分岐(ぶんき)と場面移動(いどう)

同じ物語から複数(ふくすう)の結末(けつまつ)へ進めたい場合は、場面ラベルを付(つ)け、条件(じょうけん)または入力と移動(いどう)先を同じ順番(じゅんばん)で並(なら)べます。

registerBranch=ending:score >= 3,score < 3:good-end,retry
---
sceneLabel=choice
action=branch:ending
---
sceneLabel=good-end
action=stage:Happy End
---
sceneLabel=retry
action=stage:Try Again

分岐(ぶんき)条件(じょうけん)はSB3内の実行時式評価(ひょうか)機能(きのう)で判定(はんてい)されます。外部から受け取った文字列をそのまま条件(じょうけん)へ入れず、作者が管理(かんり)する式だけを使ってください。条件(じょうけん)、キー、登場人物の一覧(いちらん)と、移動(いどう)先ラベルの一覧(いちらん)は同じ個数(こすう)にします。

10. 作品を作る手順(てじゅん)

  1. 物語を短い場面に分け、各(かく)場面の目的(もくてき)を書き出します。
  2. 背景(はいけい)、登場人物の見た目、音をTurboWarpのプロジェクトへ追加(ついか)します。
  3. 素材(そざい)名をasset、登場人物をactorとして台本の冒頭(ぼうとう)へ登録(とうろく)します。
  4. 最初(さいしょ)はポーズを使わず、背景(はいけい)、表示(ひょうじ)、せりふ、移動(いどう)、音だけで最後(さいご)まで再生(さいせい)できるようにします。
  5. 場面ごとにTMPoseモデルを用意し、TMPoseURLとposeを一つずつ追加(ついか)します。
  6. 実際(じっさい)の上演(じょうえん)環境(かんきょう)で、認識(にんしき)、音量(おんりょう)、せりふの表示(ひょうじ)時間、移動(いどう)時間を調整します。
  7. 完成(かんせい)した台本とSB3ファイルの組み合わせに版(ばん)番号と日付(ひづけ)を付(つ)けて保管(ほかん)します。

公開前チェックリスト

  • 台本をUTF-8で保存(ほぞん)した
  • 素材(そざい)名、人物名、場面ラベル、ポーズラベルの表記が一致(いっち)している
  • 一覧(いちらん)形式の引数は個数(こすう)が一致(いっち)している
  • 最初(さいしょ)から最後(さいご)まで、ポーズを成功(せいこう)させて再生(さいせい)できた
  • カメラを拒否(きょひ)した場合と、途中(とちゅう)で再(さい)読(よ)み込(こ)みした場合の案内(あんない)を確認(かくにん)した
  • 第三者の画像(がぞう)、音、物語を使う場合の利用(りよう)条件(じょうけん)を確認(かくにん)した
  • 参加(さんか)者の安全、プライバシー、撮影(さつえい)範囲(はんい)について説明(せつめい)した

11. このドキュメントの生成(せいせい)

原稿(げんこう)はdocs/index.mdで管理(かんり)しています。ビルドではVivliostyleがMarkdownをWeb Publication形式のHTMLへ変換(へんかん)し、rubyganaが設定(せってい)学年より後に学ぶ漢字へHTMLのruby要素(ようそ)を付(つ)けます。その最終(さいしゅう)HTMLをVivliostyleでA4のPDFへ組版(くみはん)します。

pnpm install
pnpm run build

既定(きてい)では小学3年生までに学ぶ漢字を既習(きしゅう)として扱(あつか)います。小学4年生までを既習(きしゅう)にする例(れい)は次のとおりです。

RUBYGANA_GRADE=4 pnpm run build

GitHub Pagesへ公開する場合は、同じ学年設定(せってい)を付(つ)けてデプロイします。

RUBYGANA_GRADEには1から6までの整数を指定できます。コード、コマンド例(れい)、プログラム名は自動ルビの対象(たいしょう)外です。自動解析(かいせき)が固有名詞(こゆうめいし)を正しく読めない場合は、docs/config.mjsのrubyOverridesへ読みを追加(ついか)してください。
