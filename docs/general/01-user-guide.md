# 紙芝居アプリ 操作説明書

対象: 紙芝居を再生する人、発表者、授業やワークショップで進行する人\
対象アプリ: TurboWarp版 紙芝居アプリ `kamishibai=3.1`

## 1. このアプリでできること

この紙芝居アプリは、テキストで書かれた紙芝居DSLファイルを読み込み、背景画像、登場人物、画面上のテキスト、セリフ、思考吹き出し、効果音、音楽、移動・切り替え演出、ポーズ認識を組み合わせて紙芝居を再生するアプリです。

普通の紙芝居のように「見る」だけでなく、途中で参加者がカメラの前でポーズを取ることで物語が進みます。`urashima.txt` では、浦島太郎がカメを助ける、カメに乗る、踊る、玉手箱を受け取る、玉手箱を開ける、といった場面がポーズ認識に組み込まれています。

3.1では、シーンに名前を付け、条件分岐やキー・タッチ入力で移動先を変えられます。フェード演出、アクター画像のループ／一回再生、ランタイム変数、画面上のテキストも台本から扱えます。

## 2. 利用に必要なもの

| 必要なもの                        | 説明                                                                   |
| --------------------------------- | ---------------------------------------------------------------------- |
| TurboWarpで実行できる紙芝居アプリ | ダウンロードページから取得できる `kamishibai-3_1a1.sb3`（暫定版）をTurboWarpで開いて使います。 |
| 紙芝居DSLファイル                 | 例: `urashima.txt`。拡張子は `.txt` を想定しています。                 |
| インターネット接続                | 画像、音声、TMPoseモデルをURLから読み込むために必要です。              |
| カメラ                            | ポーズ認識を使う場合に必要です。                                       |
| 明るい場所                        | ポーズ認識の精度が上がります。                                         |
| 発表用画面                        | 授業や発表では、プロジェクタ、電子黒板、画面共有などがあると便利です。 |

## 3. 基本的な使い方

### 3.1 アプリを起動する

1. TurboWarpで紙芝居アプリを開きます。
2. 緑の旗を押します。
3. 表紙画面とメニューが表示されます。
4. 以前に読み込んだ台本がある場合は、「前回の台本を開く」ためのボタンが表示されます。

### 3.2 新しい台本ファイルを読み込む

1. メニューにあるファイル読込ボタンを押します。
2. `.txt` の台本ファイルを選びます。
3. アプリが台本を読み込みます。
4. 画像や音声などのアセットを読み込みます。
5. 読み込みが終わると、紙芝居が始まります。

台本を一度読み込むと、ブラウザのlocalStorageに保存されます。同じブラウザであれば、次回以降は前回の台本を再生できます。

### 3.3 前回の台本を再生する

1. 緑の旗を押します。
2. 前回台本ボタンが表示されていることを確認します。
3. 前回台本ボタンを押します。
4. 保存されている台本が再生されます。

注意: localStorageはブラウザごとの保存領域です。別のブラウザ、別の端末、シークレットモードでは前回台本が表示されないことがあります。

## 4. 画面に表示される主なもの

| 表示               | 役割                                                                |
| ------------------ | ------------------------------------------------------------------- |
| 表紙画面           | 台本読み込み前、または物語終了後に表示される画面です。              |
| ファイル読込ボタン | 新しい `.txt` 台本ファイルを選ぶためのボタンです。                  |
| 前回台本ボタン     | localStorageに保存された前回の台本を再生するボタンです。            |
| Hatchling          | アセット読み込み中に表示される案内役です。読み込み数を表示します。  |
| ポーズをとろう     | ポーズ認識が必要な場面で表示されるプロンプトです。                  |
| ポーズ認識         | 現在のポーズ認識の信頼度を示す目安です。                            |
| チャージ           | 目標ポーズが一定時間認識されると増える値です。100になると成功です。 |
| 不正な台本         | 台本の書き方に問題がある場合に表示されます。                        |

## 5. 紙芝居の進み方

紙芝居はシーン単位で進みます。各シーンでは、次のような演出が自動的に順番に実行されます。

- 背景を変える
- 音楽や効果音を鳴らす
- 登場人物を表示する
- 画面上のテキストを表示・更新する
- 登場人物を移動させる
- 画像を連続切り替えしてアニメーションさせる
- セリフや思考吹き出しを表示する
- 指定秒数待つ
- ポーズ認識を行う
- 条件、キー入力、タッチ入力に応じて別のシーンへ進む

各シーンの終了時には、アクターは非表示になり、音も停止します。そのため、次のシーンで必要な登場人物や背景は、台本側で改めて表示する必要があります。

## 6. ポーズ認識シーンの操作

### 6.1 カメラを許可する

台本を再生すると、ブラウザがカメラ利用の許可を求める場合があります。ポーズ認識を使う紙芝居では、カメラを許可してください。

許可しない場合、ポーズ認識が進まず、物語が止まることがあります。

### 6.2 ポーズを取る

1. 画面に「ポーズをとろう」と表示されます。
2. カメラの前に立ちます。
3. 画面のキャラクターや進行役の指示に合わせてポーズを取ります。
4. 正しく認識されると、チャージが増えます。
5. チャージが100になると、そのポーズは成功です。
6. 複数のポーズがある場合は、次のポーズに進みます。

### 6.3 ポーズが認識されにくいとき

次の点を確認してください。

| 問題                       | 対処                                 |
| -------------------------- | ------------------------------------ |
| 暗い                       | 明るい場所に移動する、照明をつける。 |
| 体が画面からはみ出る       | カメラから少し離れる。               |
| 背景がごちゃごちゃしている | できるだけ単純な背景にする。         |
| 服と背景の色が近い         | 背景と違う色の服にする。             |
| 複数人が映っている         | ポーズを取る人だけが映るようにする。 |
| カメラが許可されていない   | ブラウザのカメラ許可を確認する。     |

## 7. 発表・授業での使い方

### 7.1 事前準備

発表前に、必ず1回は最初から最後まで再生テストをしてください。特に次の点を確認します。

- 台本ファイルが読み込めるか
- 画像が表示されるか
- 音声が鳴るか
- カメラが使えるか
- ポーズ認識が進むか
- プロジェクタや画面共有で見やすいか

### 7.2 進行のコツ

紙芝居の途中でポーズ認識が入る場合、発表者は次のように案内するとスムーズです。

> 「ここでみんなの力が必要です。画面の前でカメを助けるポーズをしてみましょう。」

ポーズ認識は、動作が大きく、止まっている時間が少し長いほど成功しやすくなります。急いで動くより、「ポーズを作って、少し待つ」のがコツです。

### 7.3 参加人数の考え方

ポーズ認識は基本的に、カメラに映っている人の姿勢をもとに判断します。大人数が一度に映ると認識が不安定になることがあります。授業では、代表者が前に出てポーズを取る形式が扱いやすいです。

## 8. リハーサル・開発者用キー操作

発表練習や台本の動作確認では、次のキー操作が使えます。

| キー         | 動作                                 | 用途                             |
| ------------ | ------------------------------------ | -------------------------------- |
| スペースキー | 現在のポーズ認識をスキップ           | ポーズ認識を手早く通過したいとき |
| 右矢印キー   | 現在シーンの残りアクションをスキップ | 演出確認中に先へ進めたいとき     |
| 下矢印キー   | 現在シーンをスキップ                 | シーン単位で確認したいとき       |

本番中に誤って押すと演出が飛ぶため、発表用PCではキーボード操作に注意してください。

## 9. よくあるトラブルと対処

### 9.1 「不正な台本」と表示される

台本の文法に誤りがある可能性があります。

確認すること:

- 先頭に `kamishibai=3.1` があるか
- `key=value` の形式になっているか
- 未対応のコマンド名を書いていないか
- `action=` の区切り `:` が不足していないか
- `---` の位置が正しいか
- 分岐先の `sceneLabel` が存在するか

### 9.2 画像や音が読み込まれない

アセットURLが間違っている、URL先にアクセスできない、ネットワークが使えない、ブラウザが読み込みを拒否している可能性があります。

確認すること:

- URLをブラウザで直接開けるか
- ファイル名の大文字・小文字が一致しているか
- 画像や音声が公開URLに置かれているか
- ネットワークが安定しているか

### 9.3 紙芝居が途中で止まる

多くの場合、ポーズ認識待ち、音声の再生待ち、または長い `wait` の途中です。

確認すること:

- 「ポーズをとろう」が表示されていないか
- `sound` で長い音声を再生していないか
- `wait` の秒数が大きすぎないか

### 9.4 セリフが想定通りに出ない

台本内の `:` はアクションの区切り文字として使われます。セリフ本文に `:` を入れると、意図しない分割になる場合があります。

対処:

- セリフ内では `：` のような全角記号を使う
- `=` も台本コマンドの区切りに使われるため、本文ではなるべく避ける
