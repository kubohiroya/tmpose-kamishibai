## 0. この教材と体験会について

### 0.1 この教材の一次配布元 {#0.1-この教材の一次配布元}

AI紙芝居アプリおよびこの教材は、久保裕也（千葉商科大学　総合政策学部　教授）による研究プロジェクトの成果物です。正式版の配布は、GitHub Pages [https://kubohiroya.github.io/tmpose-kamishibai/](https://kubohiroya.github.io/tmpose-kamishibai/) で行うものとします。

AI紙芝居アプリも、ここから遊べるようにします。(準備中）

### 0.2 この教材の狙い

このAI紙芝居アプリでは、サンプル台本として「浦島太郎」の物語を利用できます。ここでは、浦島太郎が竜宮城に到着したときに、浦島太郎自身が利用者といっしょにダンスをするようになっていますが、このワークショップでは、台本を書き換えるなどして、乙姫様が利用者といっしょにダンスをするという内容に変更してみましょう。

乙姫様にどんなダンスポーズをさせるかを考え、棒人間の絵を描き、それをもとにAIに乙姫様の絵を描かせてみましょう。さらに、Teachable Machineでダンスポーズを認識できるモデルのトレーニングをしてみましょう。最後に、紙芝居の台本を更新し、AIによる成果物をとりまとめて、作品全体を遊べるようにしてみましょう。

### 0.3 この体験会が想定する参加者

「小学生の児童およびその保護者のペア」で参加されることを標準的な形として想定しています。

* プログラミング経験（Scratchなど）は問いません
* AI利用経験は問いません

また、次のことを確認させてください。

* マウスやキーボードの操作ができますか？
* EdgeなどのWebブラウザの操作ができますか？
* Windowsにおけるフォルダやファイルの概念を理解していますか？

わからないという人は、この会を通じてスタッフがサポートしますので、安心してください。

### 0.4 この体験会で学ぶこと

この体験会では、

* 画像づくりAI
* ポーズ認識AI
* TurboWarpによる紙芝居プログラミング

といった技術を組み合わせて、

**「AI技術を応用して、紙芝居の物語の中に参加する仕組み」**

を親子で作ります。楽しんでください！

#### 1.1.2 カメラの使用の準備をするとき

液晶ディスプレイの上に、カメラが載せられているのを確認してください。
この体験会の中では、小学生の方が座っている席のパソコンのカメラを使用します。
保護者の方が座っている席のパソコンのカメラは予備として、基本的に使用しません。

この体験会の途中で、講師から「カメラの使用を準備してください」と指示される場合があります。そのときには、小学生の方が座っている席の液晶ディスプレイを台座から回転させて、「紺色の矢印」の方向に向けてください。

カメラでの撮影内容に他の親子が写り込まないように、カメラの向きに注意してください。

#### 1.1.3 カメラの使用をするとき

「カメラの使用をしてください」と指示する場合には、小学生の方または保護者が席を離れてカメラの前でポーズを取り、ペアのもう一人がパソコンを操作してください。

カメラの前でポーズを取るときには、周囲の人や物にぶつからないように注意してください。

## 2. AIに乙姫様を描いてもらおう

まずはじめに、乙姫様を踊らせる＝あなたが踊りたいダンスのポーズの絵を描きましょう。あなたは、どんなポーズで乙姫様を踊らせたいですか？　あなたは、どんなポーズでダンスをしたいですか？

### 2.1 ダンスのポーズを考えて棒人間の絵で下書きする {#2.1-ダンスのポーズを考えて棒人間の絵で下書きする}

これから、ダンスのポーズの絵を描くにあたって、絵の上手下手を気にしなくてよいです。あなたには、「棒人間」の絵を描いてもらいます。

#### 2.1.1 手書きで棒人間の絵を描く

メモ用紙に、

* ポーズの様子を示した「棒人間」の絵を手書きで描いてください。
* ポーズの順番に応じた「番号」を記入してください。

身体全体の大きな動きがわかるように描いてください。細部をリアルに描く必要はありません。

こうしたメモを最低2枚、最大4枚程度作成してください。

その際、できるかぎり、それぞれのポーズで、左右の手の角度、折り曲げ・伸ばしで違いがはっきりわかるようなものにしてください。似たようなポーズ、他のポーズとまぎらわしいポーズは、最初から避けるようにしてください。とくに、下半身は認識しにくいので、下半身だけが異なるようなポーズにはしないようにしましょう。

たとえば、次のようなポーズの絵を描けばよいということです。

うまく描けなかったら、別の紙に描き直しをしてください。

![][image12]①

### 2.2  パソコンで棒人間の絵を描く

#### 2.2.1 「配布資料」フォルダを開く

Windowsのデスクトップに、この体験会の「配布資料」として、「20260801」（今日の日付：2026年8月1日)という名前のフォルダのアイコンが配置されています。以降、このフォルダを「配布資料」フォルダと呼ぶことにします。

このアイコンをダブルクリックすると、その内容が表示されます。

ここには、次のような4つのフォルダがあります。![][image13]

| フォルダ名 | 内容の説明 |
| :---- | :---- |
| draft | 白紙の画像が4つ入っています。アプリで編集して「棒人間」の画像として保存します。これらは、生成AIで画像を生成するときの素材として使います。 |
| generated | 体験会の開始時には何も入っていません。体験会を通じて生成AIで作成した画像を保存します。 |
| generated-sample | 乙姫様のダンスポーズのサンプル画像があらかじめ入っています。体験会中に生成AIがうまく動作しない場合に使います。 |
| master | 浦島太郎や乙姫様の立ち絵の画像です。これらは、生成AIで画像を生成するときの素材として使います。 |

フォルダの中を表示するときには、用途に応じて、そのファイルの内容を「アイコン」として表示したいという場合、ファイル名だけで表示したいという場合などがあります。このように、フォルダの中を表示する方法を切り替えるためには、フォルダ画面上部のメニュー中の「表示」ボタンをクリックして、表示方式を選択できます。ここでは、「中アイコン」「詳細」の2つを試してみておいてください。

| p1.png p2.png p3.png  (作成した場合) p4.png  (作成した場合) |

1. 「ダウンロード」フォルダで、「iloveimg-background-removed」フォルダのアイコンを右クリックします。
2. 「ゴミ箱」（削除）のアイコンをクリックします。

## 6. 台本ファイルの修正をしよう

### 6.1 台本ファイルの編集を開始する

配布資料フォルダの中の、紙芝居の台本ファイル「my-urashima.txt」ファイルをダブルクリックし、「メモ」アプリで開いてください。

＜⭐️ここのスクリーンショットを日本語UIのものに差し替える＞

![][image59]

台本ファイルの中では、行ごとに、台本を構成するさまざまな設定が記述されています。

行の1文字目に記号「\#」がつけられていると、その行の設定は無効化されます。

my-urashima.txtファイルでは、さまざまな状況に対応した設定の候補を、行の1文字目に記号「\#」をつける仕組みを用いて、あらかじめ無効化した形で書いて並べてあります。

my-urashima.txtファイルを修正する際には、有効化したい設定の行の1文字目の「\#」を削除することで、設定をカスタマイズすることができます。

行が「---」という内容のものは、物語のシーンの区切りを表しています。最初の「---」より前は、物語の全てのシーンで共通して利用される「アセット」に関する設定などを行います。

### 6.2 モデルのURLを設定する

AI紙芝居アプリでは、それぞれのシーンで、カメラ画像から身体のポーズを認識して、それに応じて物語を進めるという仕組みをつくることができます。そのためには、5章で行ったように、事前にカメラの画像でポーズを撮影し、これを認識するためのAIモデルTeachable Machineのトレーニングをし、その成果物であるモデルをクラウドにエクスポートして共有可能にしておきます。ここでいう「モデルのURL」とは、5.6節でモデルをクラウドにアップロードして共有可能にしておいたもののURLのことです。

たとえば、シーン3「竜宮城到着」のモデルのURLの設定は、my-urashima.txtの123行目（\# scene 3という行の下の行）にあります。この台本では、もともと浦島太郎がダンスをするポーズを認識するモデルのURLが設定されています。

それでは、これを、あなたが作った乙姫様のダンスのポーズを認識するように修正しましょう。5.6節でリンクをコピーしておいたものを、123行目での、TMPoseURL=以下の部分に貼り付けして既存の内容を置き換えてください。

| \--- \# scene 3 TMPoseURL=<https://teachablemachine.withgoogle.com/models/…>  |
| :---- |

【間違った内容を貼り付けしていないか、=記号などを消してしまっていないか確認してください】

### 6.3 アセットを設定する

ここでいう「アセット」とは、このAI紙芝居アプリにおいて、背景画像、コスチューム画像、効果音、BGMなどの素材を統合した呼び名です。これは、ScratchやTurboWarpの標準的な用語・概念ではなく、TurboWarp機能拡張による[turbowarp-asset-manager](https://github.com/kubohiroya/turbowarp-asset-manager)で実現される用語・概念です。

アセットを設定するためには、その行を asset= で始めます。カンマ区切りでの1つめの値がアセット名、2つめの値がアセットのアドレス値を表します。

my-urashima.txtファイルの22行目から25行目では、Scratchのスプライトで Princess という名前のものについてのコスチュームを、アセットとして設定しています。

ここでは、22行目の設定がどのようになっているのかを説明します。

**asset=p1,costume:Princess**

| 設定行の部分 | 意味 |
| ----- | ----- |
| asset= | この行で「アセット」を設定することを宣言する |
| p1 | アセット名は「p1」であることを表します。 |
| , | 区切り文字「,」はアセット名には含むことができないルールとしており、アセット名の終端を表します。 |
| costume: | アセットのアクセス先は「costume:」であることを表します。 |
| Princess | コスチュームを探す際には「Princess」という名前のスプライトの中から、アセット名(p1)と同じ名前のコスチュームを探します。 |

それではここで、あなたが作成した乙姫様の画像をアセットとして設定しましょう。

アセットの設定を有効化したい行の内容について、行頭の\#記号を削除してもらいます。その行の設定により、アセット名として、Princessというスプライト名の同名のコスチューム画像を登録することができるのです。

【必要な行についてだけ、行頭の\#を削除するという方針について確認してください】

| \#asset=p1,costume:Princess \#asset=p2,costume:Princess \#asset=p3,costume:Princess \#asset=p4,costume:Princess |
| :---- |

以下の 6.3.1 、6.3.2、6.3.3 節は、場合ごとの設定内容を示したものです。このうちいずれかひとつの形で設定を実施してください。

#### 6.3.1 ポーズ2つを作成した場合の設定

「p1」をアセット名として、Princessというスプライト名の「p1」という名前のコスチューム画像を登録します。

「p2」をアセット名として、Princessというスプライト名の「p2」という名前のコスチューム画像を登録します。

| asset=p1,costume:Princess asset=p2,costume:Princess \#asset=p3,costume:Princess \#asset=p4,costume:Princess |
| :---- |

#### 6.3.2 ポーズ3つを作成した場合の設定

「p1」をアセット名として、Princessというスプライト名の「p1」という名前のコスチューム画像を登録します。

「p2」をアセット名として、Princessというスプライト名の「p2」という名前のコスチューム画像を登録します。

「p3」をアセット名として、Princessというスプライト名の「p3」という名前のコスチューム画像を登録します。

| asset=p1,costume:Princess asset=p2,costume:Princess asset=p3,costume:Princess \#asset=p4,costume:Princess |
| :---- |

#### 6.3.3 ポーズ4つを作成した場合の設定

「p1」をアセット名として、Princessというスプライト名の「p1」という名前のコスチューム画像を登録します。

「p2」をアセット名として、Princessというスプライト名の「p2」という名前のコスチューム画像を登録します。

「p3」をアセット名として、Princessというスプライト名の「p3」という名前のコスチューム画像を登録します。

「p4」をアセット名として、Princessというスプライト名の「p4」という名前のコスチューム画像を登録します。

| asset=p1,costume:Princess asset=p2,costume:Princess asset=p3,costume:Princess asset=p4,costume:Princess |
| :---- |

### 6.4 アクターのポーズを設定する

ここでいう「アクター」とは、このAI紙芝居で表示されるキャラクターを表す呼び名です。「浦島太郎」「亀」「乙姫様」などの全てのキャラクターは、「アクター」です。このAI紙芝居アプリでは、Scratchの標準的な用語・概念であるスプライトを直接使わずに、「Actor」という英語名の特殊なスプライトのクローン（複製）し、これを識別できるようにスプライト変数「アクター名」を用いて名前をつけて管理しています。

アクターは、「アクション」をすることができます。台本ファイルの中で行頭が「action=アクター名:」で始まる設定のことを「アクターアクション」と呼んでいます。

ここでは、my-urashima.txtファイルの140行目で「Princess」という名前のアクターに、「pose」というアクションをさせる例を見てみましょう。

**action=Princess:pose:p1,p2,p1,p2:pose1,pose2,pose1,pose2:Drum Funky,Drum Funky,Drum Funky,...**

| 設定行の部分 | 意味 |
| ----- | ----- |
| action= | この行は「アクション」を設定するものであることを宣言します |
| Princess: | この行はアクター名「Princess」の「アクターアクション」であることを宣言します |
| pose: | この行はアクターアクション「pose」を設定することを宣言します |
| p1,p2,p1,p2 | アセット名「p1」「p2」「p1」「p2」の順番で示されたコスチュームが用いられて、アクターの見た目を変化させることを指定します。 |
| pose1,pose2,pose1,pose2 | Teachable Machineのポーズのクラス名（ポーズ名）をカンマ区切りで列挙します。pose1,pose2,pose1,pose2の順番でポーズを認識することを指定します。 |
| Drum Funky,Drum Funky,Drum Funky,Drum Funk | アセット名「Drum Funky」「Drum Funky」「Drum Funky」「Drum Funky」の順番で示された効果音が用いられ、ポーズの認識に合わせて音が鳴らされることを指定します。 |

それでは、my-urashima.txtファイルの140行目-142行目の3行のうちで、設定を有効化したい行の内容について、行頭の\#記号を削除してください。

【必要な行についてだけ、行頭の\#を削除するという方針について確認してください】

| \#action=Princess:pose:p1,p2,p1,p2:pose1,pose2,pose1,pose2:Drum Funky,Drum Funky,Drum Funky,... \#action=Princess:pose:p1,p2,p3,p1,p2,p3:pose1,pose2,pose3,pose1,pose2,pose3:Drum Funky,Drum Funky,... \#action=Princess:pose:p1,p2,p3,p4,p1,p2,p3,p4:pose1,pose2,pose3,pose4,pose1,pose2,pose3,pose4:Drum... |
| :---- |

以下の 6.4.1、 6.4.2、 6.4.3 節は、場合ごとの設定内容を示したものです。このうちいずれかひとつの形で設定を実施してください。

#### 6.4.1 ポーズ2つを作成した場合の設定

3行のうち1行目、アセット名部分に「p1」「p2」が書かれている行の行頭の\#記号を削除してください。ここでは、pose1,pose2のポーズ2つが2回繰り返されます。

| action=Princess:pose:p1,p2,p1,p2:pose1,pose2,pose1,pose2:Drum Funky,Drum Funky,Drum Funky,... \#action=Princess:pose:p1,p2,p3,p1,p2,p3:pose1,pose2,pose3,pose1,pose2,pose3:Drum Funky,Drum Funky,... \#action=Princess:pose:p1,p2,p3,p4,p1,p2,p3,p4:pose1,pose2,pose3,pose4,pose1,pose2,pose3,pose4:Drum... |
| :---- |

#### 6.4.2 ポーズ3つを作成した場合の設定

3行のうち2行目、アセット名部分に「p1」「p2」「p3」が書かれている行の行頭の\#記号を削除してください。ここでは、pose1,pose2,pose3のポーズ3つが2回繰り返されます。

| \#action=Princess:pose:p1,p2,p1,p2:pose1,pose2,pose1,pose2:Drum Funky,Drum Funky,Drum Funky,... action=Princess:pose:p1,p2,p3,p1,p2,p3:pose1,pose2,pose3,pose1,pose2,pose3:Drum Funky,Drum Funky,... \#action=Princess:pose:p1,p2,p3,p4,p1,p2,p3,p4:pose1,pose2,pose3,pose4,pose1,pose2,pose3,pose4:Drum... |
| :---- |

#### 6.4.3 ポーズ4つを作成した場合の設定

3行のうち3行目、アセット名部分に「p1」「p2」「p3」「p4」が書かれている行の行頭の\#記号を削除してください。ここでは、pose1,pose2,pose3,pose4のポーズ4つが2回繰り返されます。

| \#action=Princess:pose:p1,p2,p1,p2:pose1,pose2,pose1,pose2:Drum Funky,Drum Funky,Drum Funky,... \#action=Princess:pose:p1,p2,p3,p1,p2,p3:pose1,pose2,pose3,pose1,pose2,pose3:Drum Funky,Drum Funky,... action=Princess:pose:p1,p2,p3,p4,p1,p2,p3,p4:pose1,pose2,pose3,pose4,pose1,pose2,pose3,pose4:Drum... |
| :---- |
